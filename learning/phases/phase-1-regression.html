The Big Idea

You have N monthly expense totals. Draw the single best line through them.
₹ Amount │ ● ← actual month │ ● / │ ● → best line / │ ● / │ / ● └─────────────────────────── time
Once you have the line, extend it forward → that's forecasting. The same algorithm trains every neural network in existence — just with many more weights.

Conceptual Foundations

Scalar vs Vector
A single number is not a vector. A list is.
Linear AlgebraPhase 1
3262 ← scalar (one number) [3262, 1250, 20260315, 4] ← vector (a whole row) [1250, 480, 3200, 750, 950, ...] ← vector (a whole column, N elements)
Vector = ordered list of numbers. Length is fixed. Order matters.
A vector is one row OR one column of your expenses table. For Phase 1, the amount column across many months IS your y-vector.
A scalar has no dimensions; a vector lives in n-dimensional space. ML datasets are matrices of feature vectors (rows) plus a target vector (the label column).
Dot Product as a "Weighted Combine"
Sum, mean, weighted average, and a regression prediction are all the same operation.
Linear AlgebraPhase 1Phase 5
months = [1, 2, 3] expenses = [40, 50, 60] months · expenses = 1×40 + 2×50 + 3×60 = 320
a · b = a₁·b₁ + a₂·b₂ + ... + aₙ·bₙ
Same operation, different weight vectors → different meanings: expenses · [1, 1, 1] = 150 ← SUM expenses · [1/3, 1/3, 1/3] = 50 ← MEAN expenses · [w₁, w₂, w₃] = ? ← WEIGHTED AVERAGE / PREDICTION
The dot product is the universal "combine these numbers, weighting them however I want" operation. Linear regression's only job: pick the BEST weights.
Sum, mean, dot product, matrix multiplication, neural network forward pass, and transformer attention are all built from one operation: element-wise multiply then sum.
Features vs Weights
The leap from regular programming to machine learning
Core ML ConceptAll Phases
What it isWho decides
FeaturesThe inputs you measure → e.g. [month, 1]YOU — by deciding what to measure
WeightsNumbers multiplied with features → e.g. [slope, intercept]The algorithm, by minimising loss on your data
Regular Programming Machine Learning ─────────────────── ──────────────── You: Write the rule Provide (input, output) examples if month > 9: predict 60000 (1, 45k), (2, 50k), (3, 48k), ... Output: Constants you chose Constants the algorithm chose (slope, intercept) → the trained model
In regular code you write the rule. In ML you give examples and the algorithm discovers the rule. For Phase 1, "the rule" is the two numbers (slope, intercept) that minimise SSE.
A pretrained LLM is a 70-billion-weight vector. Nobody set those values by hand — gradient descent did, on terabytes of text.
In ML, you choose the model architecture and feature representation; the algorithm finds the parameter values. This separation is what makes ML scale to problems where rules are too complex to hand-code.
The Bias Trick — why a constant "1" sneaks in
Reformulating the intercept so every prediction is a clean dot product
Linear AlgebraPhase 1Phase 5
Naive: prediction = slope × month + intercept Reframed: prediction = slope × month + intercept × 1 ↑ fake feature, always 1 Dot product form: prediction = [slope, intercept] · [month, 1] └── weights ───┘ └ features ┘
Every ML prediction in the universe has shape: weights · features
If you skip the "1", then at month=0 the prediction is slope×0 = 0 — you'd have no baseline. The constant 1 lets the intercept survive the dot product.
The bias term is a fake feature whose value is always 1. It lets us write any affine function (Wx + b) as a single dot product. Every neural net layer uses this trick.

The Line Equation

Where does y = mx + b come from?
Derived from the definition of "straight" — not assumed
GeometryPhase 1
A line is the UNIQUE curve whose slope is the same everywhere. If the slope changed, the curve would bend → not straight. y │ (x₂, y₂) │ ● │ ╱│ │ ╱ │ rise = y₂ − y₁ │ ╱ │ │ (x₁,y₁) │ │ ●─────┘ │ ╲ run = x₂ − x₁ └──────────────────── x For ANY two points on the same line: slope = (y₂ − y₁) / (x₂ − x₁) = constant = m
Derivation — pick reference point (0, b) where the line crosses the y-axis. Pick a general point (x, y). Apply the constant-slope property: (y − b) / (x − 0) = m y − b = m · x y = m · x + b ✓
y = m·x + b ↔ "start at height b. Move x right. Line went up by m·x. You're now at m·x + b."
SymbolNameMeaning
mSlopeHow fast y changes per unit of x
bInterceptValue of y when x = 0
xInpute.g. month index 1, 2, 3, ...
yOutputWhat the line predicts
Why a line is the right starting model: it's the simplest non-trivial relationship. Rule #1 of ML — start with the simplest model that could work. Anything fancier only earns its keep if it beats the line.
Prediction (ŷ) vs Actual (y)
The hat-notation matters — they're different numbers
NotationAll Phases
y_i ← actual observed value for point i (from Supabase) ŷ_i "y-hat" ← what the MODEL predicts for point i ŷ_i = m · x_i + b The GAP between them is what we want to minimise: error_i = y_i − ŷ_i
Bare y = truth. y-hat = our guess. The whole job of training is to make ŷ approach y across all data points.
From y = mx + b to SSE = Σ(y − m·x − b)²
Three substitution links — every symbol has a name
DerivationPhase 1
Link 1 — apply the line to each data point: ŷ_i = m · x_i + b Link 2 — error per point = actual minus predicted (substitute ŷ_i): error_i = y_i − ŷ_i = y_i − (m · x_i + b) = y_i − m · x_i − b ← just expanded the parentheses Link 3 — square (so signs don't cancel + smooth math) and sum across N points: SSE = Σ (y_i − m · x_i − b)²
SSE = Σ ( yᵢ − m·xᵢ + b )² ↑ ↑ actual predicted by the line
Every symbol inside the parentheses came directly from the line equation. m·x_i + b is just "what the line predicted at point i". Subtracting it from y_i gives the error. Squaring and summing gives a score.

The Loss Function

SSE — Sum of Squared Errors
A single number that scores how bad a candidate line is
Loss FunctionPhase 1
SSE = Σ (y_i − ŷ_i)² ← sum of squared errors across all N points
Worked example — candidate line m=2000, b=40000: Data point Prediction Error Squared (1, ₹45k) → 2000×1+40000=42k +3 9 (2, ₹50k) → 2000×2+40000=44k +6 36 (3, ₹48k) → 2000×3+40000=46k +2 4 ───── SSE = 49 A different line gives a different SSE. The line with the SMALLEST SSE wins.
Is SSE=49 good or bad? Meaningless on its own — depends on the scale of y and the number of points. That's why we ALSO compute R² (a 0-to-1 normalised score) — coming up next session.
SSE / MSE (Mean Squared Error = SSE/N) are the standard regression losses. For classification you'd use cross-entropy; for robust regression, Huber loss. The model + loss + minimiser triple defines every ML system.
Why Squared Errors?
A loss function must satisfy three properties — squaring is the cleanest fit
Math Intuition
PropertyWhy it mattersHow squaring delivers it
Penalises wrong predictionsBigger miss = bigger scoree² grows with |e|
Treats +/− errors equally+500 isn't better than −500Squaring kills the sign
Smooth (no kinks)So calculus can find the minimumx² smooth everywhere; |x| has a kink at 0
Punishment grows quadratically: a ₹10k error is 100× worse than ₹1k, not 10×.
MAE (Mean Absolute Error) is also valid and robust to outliers — but no clean closed-form solution because of the kink in |x| at 0. Phase 1 uses SSE because it gives us the exact formula.

Finding the Best Line

The Loss Bowl + Why Derivatives
SSE is a function of (m, b). The minimum is where derivatives equal zero.
CalculusPhase 1Phase 5
SSE depends on two unknowns (m, b) → 3D bowl shape: SSE │ ╱╲ │ ╱ ╲ ← high SSE = bad line │ ╱ ╲ │ ╱ ● ╲ ← bottom of bowl = best (m*, b*) │ ╱ ╲ └────────────── (m, b) plane
Finding the minimum of any function f(x): f(x) │ ╲ ╱ │ ╲ ● ╱ ● = minimum (derivative = 0) │ ╲ ╱ ╲ ╱ │ ╲ ╱ ╲ ╱ │ ╲╱ ╲ ╱ └──────────────────── x slope < 0 slope = 0 slope > 0 (downhill) (bottom) (uphill)
Recipe: set ∂SSE/∂m = 0 AND ∂SSE/∂b = 0 → two equations, two unknowns
Imagine standing on a hillside in fog. The slope of the ground tells you which way is downhill. Where the ground is flat in every direction — you're at the bottom. Calculus formalises this.
Minimising a loss function = setting its gradient (vector of partial derivatives) to zero. For convex losses like SSE on a linear model, this has a closed-form solution. For neural networks the loss is non-convex, so we walk downhill iteratively (gradient descent).
The Full Derivation — where the formulas come from
Two simple moves: take the derivative, set it to zero. Read once, never wonder again.
CalculusPhase 1
Starting point: SSE(m, b) = Σ (yᵢ − m·xᵢ − b)²
Step 1 — derivative w.r.t. b
∂SSE/∂b = Σ 2·(yᵢ − m·xᵢ − b)·(−1) = −2 · Σ (yᵢ − m·xᵢ − b) Set to zero: Σ (yᵢ − m·xᵢ − b) = 0 Σ yᵢ − m·Σ xᵢ − N·b = 0 ← N copies of b Divide by N (take means): ȳ − m·x̄ − b = 0
→ b = ȳ − m·x̄ ✓ INTERCEPT FORMULA
Meaning: the best line is FORCED to pass through (x̄, ȳ) — the centre of mass of your data. Once slope is known, intercept is determined.
Step 2 — derivative w.r.t. m
∂SSE/∂m = Σ 2·(yᵢ − m·xᵢ − b)·(−xᵢ) ← inner derivative is −xᵢ now = −2 · Σ xᵢ·(yᵢ − m·xᵢ − b) Set to zero: Σ xᵢ·yᵢ − m·Σ xᵢ² − b·Σ xᵢ = 0 ... (★)
Step 3 — substitute b from Step 1, solve for m
Plug b = ȳ − m·x̄ into (★): Σ xᵢ·yᵢ − m·Σ xᵢ² − (ȳ − m·x̄)·Σ xᵢ = 0 Σ xᵢ·yᵢ − ȳ·Σ xᵢ = m·Σ xᵢ² − m·x̄·Σ xᵢ Σ xᵢ·(yᵢ − ȳ) = m · Σ xᵢ·(xᵢ − x̄) Algebra trick: Σ x̄·(yᵢ − ȳ) = 0 and Σ x̄·(xᵢ − x̄) = 0, so we can replace xᵢ with (xᵢ − x̄) on both sides: Σ (xᵢ − x̄)·(yᵢ − ȳ) = m · Σ (xᵢ − x̄)² Solve: Σ (xᵢ − x̄)·(yᵢ − ȳ) cov(x, y) m = ──────────────────────── = ─────────── Σ (xᵢ − x̄)² var(x)
→ m = cov(x, y) / var(x) ✓ SLOPE FORMULA
For convex losses on linear models, ∇Loss = 0 has a closed-form solution. For neural nets the gradient equation is too hairy to solve algebraically, so we walk downhill iteratively. Same principle: find where the gradient is zero.
Closed-form solutions like this are why linear models still dominate banking, insurance, and any regulated industry — every prediction is reproducible from a tiny, auditable formula.
The Universal ML Recipe
Every algorithm in this learning plan follows this exact 3-step pattern
Mental ModelAll Phases
Memorise this
1. Pick a model → here: ŷ = mx + b 2. Pick a loss → here: SSE 3. Minimise the loss ├── Closed-form → calculus → exact formulas (linear regression) └── Iterative → gradient descent (every other model)
Linear regression is the ONLY model in this plan with a closed-form answer. Logistic regression, decision trees, neural nets, transformers — all use gradient descent because the math gets too gnarly for closed-form. But the recipe is the same: model + loss + minimise.
When you read an ML paper, the first three things to identify: the model architecture, the loss function, and the optimiser. That's the whole system in a single sentence.
Slope Formula — Plain English Meaning
m = cov(x,y) / var(x) is not magic — it's forced by minimising SSE
StatisticsPhase 1
Σ (xᵢ − x̄)(yᵢ − ȳ) cov(x, y) m = ──────────────────── = ────────── Σ (xᵢ − x̄)² var(x)
TermNameWhat it measures
cov(x, y)CovarianceHow much x and y move together
var(x)VarianceHow much x spreads out on its own
Slope = (how much x and y move together) ÷ (how much x moves)
cov > 0 → x and y rise together → slope positive cov < 0 → x rises while y falls → slope negative cov ≈ 0 → y ignores x → slope ≈ 0 Intercept formula: b = ȳ − m·x̄ → The best line ALWAYS passes through (x̄, ȳ) — the centre of mass of your data. → Once you know the slope, the intercept is forced.
For your monthly expenses, x = month index (1..N), y = total spend. Slope tells you how much your spending changes per month. R² (next) tells you whether the line is actually trustworthy.

Evaluating the Fit

R² — Is the Line Actually Worth Trusting?
SSE on its own is unreadable. R² normalises it to a 0-to-1 score you can interpret.
StatisticsPhase 1Phase 3
SSE Σ (yᵢ − ŷᵢ)² R² = 1 − ───── = 1 − ────────────── SST Σ (yᵢ − ȳ)² SSE = sum of squared errors from YOUR line SST = sum of squared errors from the WORST-CASE model (just predict the mean every time)
Interpretation: R² = 1.0 → perfect prediction (every point on the line) R² = 0.7 → your line explains 70% of the variance — good R² = 0.3 → weak trend — line barely better than predicting the mean R² = 0.0 → your line is no better than just shouting "ȳ" every time R² < 0 → your line is WORSE than the mean — model is broken
R² answers: "How much of the wiggling in y did my line actually explain?" 0% means none — y is random as far as your features are concerned. 100% means the line nailed it.
For your monthly expenses with "noisy" trend, expect R² somewhere between 0.3 and 0.7. Below 0.3, don't trust the forecast — the line isn't capturing enough signal to project forward safely.
R² is also called the coefficient of determination. It compares your model's error against a baseline (predict the mean). Always report R² alongside any regression — without it, slope and intercept are meaningless.

Beyond Closed-Form

Gradient Descent — How Neural Nets Find Weights
When the math is too hairy for closed-form, walk downhill instead
OptimisationPhase 1Phase 5
Linear regression gives us the slope/intercept in ONE shot via the closed-form formula (cov/var, etc.). Neural networks have millions of weights and a non-convex loss surface → no closed form possible. Instead: walk downhill iteratively. The algorithm: 1. start with RANDOM weights (slope, intercept, ... whatever) 2. loop: a. compute current SSE on the data b. compute the GRADIENT — direction of steepest uphill c. step a tiny amount in the OPPOSITE direction (downhill) d. repeat until SSE stops improving
"gradient" = vector of partial derivatives = (∂SSE/∂m, ∂SSE/∂b, ...) Walking the OPPOSITE direction is called "gradient descent".
SSE │ ╲ ╱ │ ╲ ←─ start here (random) ╱ │ ╲ ╱ │ ●─→ ● ─→ ● ─→ ● ─→ ● ─→ ● ● = minimum │ walk downhill └───────────────────────────────── weight
Imagine standing on a foggy hillside. You can't see the valley — but you can feel which way the ground slopes. Step downhill. Repeat. Eventually you reach the bottom. Same idea, in millions of dimensions.
Linear regression: closed-form (cov/var). Logistic regression: gradient descent (no closed form because of the sigmoid). Neural networks: gradient descent + backpropagation (chain rule applied through layers). The RECIPE is the same — only step 3 differs.
For convex losses on linear models, gradient descent and closed-form give the same answer. For non-convex losses (neural nets), there's no closed form and gradient descent may converge to a local minimum, not the global one. Choice of learning rate, initialisation, and optimiser (SGD, Adam) all matter.
