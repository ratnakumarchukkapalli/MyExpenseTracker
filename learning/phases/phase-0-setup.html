Vector
A list of numbers representing one thing

Your Swiggy receipt. One purchase described by multiple values. Convert each to a number → that list IS a vector.
Google Search converts your query into a vector. Results whose vectors are closest get ranked higher.
A vector is an ordered list of numbers representing a point in n-dimensional space. In ML, one row of a dataset is called a feature vector.

Dot Product
Multiply element-by-element, then sum. The core operation of ALL ML.
Linear AlgebraPhase 1Phase 5Phase 6
Step 1 — multiply element-by-element: a = [ 1, 2, 3 ] × × × b = [ 4, 5, 6 ] = = = [ 4, 10, 18 ] Step 2 — sum: 4 + 10 + 18 = 32
a · b = a[0]×b[0] + a[1]×b[1] + ... + a[n]×b[n]
Linear regression IS a dot product: features = [month, prev_spend, season] weights = [ w1, w2, w3 ] prediction = features · weights The algorithm finds the best weights vector — that's its only job. Similarity rule: a·b = large → vectors are similar (same direction) a·b = 0 → nothing in common (perpendicular)
Netflix: your_taste · movie_profile = match score. Higher dot product = better match. Every recommendation engine works this way.
Dot product is the fundamental operation in ML. Linear regression, neural network layers, and transformer attention are all dot products. It measures alignment between two vectors.
Mean & Median
Two ways to describe the centre of your data
StatisticsPhase 1Phase 2
Mean
mean = sum / count = (100+200+300+400+500) / 5 = 300
Sensitive to outliers. One large EMI pulls it up.
Median
Sort → pick middle value [100, 200, 300, 400, 500] median = 300
Ignores outliers. Use when data is skewed.
Mean = ₹3,999 vs Median = ₹619. EMI/CC payments pull mean up. Median tells the real story — most day-to-day expenses are under ₹619.
5 friends' salaries: ₹3L, ₹4L, ₹5L, ₹6L, ₹82L. Mean = ₹20L (looks rich). Median = ₹5L (actual reality). One outlier destroyed the mean.
When mean >> median, data is right-skewed. For skewed distributions, median is a better measure of central tendency. Always check both.
Variance & Standard Deviation
How spread out is the data from the centre?
StatisticsPhase 2Phase 5
Given: [100, 200, 300, 400, 500], mean = 300 Step 1 — distance from mean: 100-300=-200, 200-300=-100, 300-300=0, 400-300=+100, 500-300=+200 Step 2 — square each: 40000 + 10000 + 0 + 10000 + 40000 = 100000 Step 3 — average: variance = 100000 / 5 = 20000 (in rupees²) Step 4 — square root to get back to rupees: std dev = sqrt(20000) = 141
variance = Σ(xᵢ - mean)² / n std dev = √variance
68-95-99 rule (critical for Phase 2 anomaly detection): 68% of data falls within mean ± 1 std dev 95% of data falls within mean ± 2 std dev 99.7% of data falls within mean ± 3 std dev Anything beyond ±2 std dev = likely anomaly (top 5%)
Mean=₹3,999, Std=₹13,301. Std larger than mean = extreme spread. Data is very skewed due to EMI/CC entries.
Two cricket batsmen both average 50. Batsman A always scores 45-55 (low std dev = consistent). Batsman B scores 0 or 100 randomly (high std dev = unpredictable).
Std dev = √variance, expressed in original units. Used in Z-score (Phase 2), confidence intervals, and weight initialisation. High std dev relative to mean signals skewed data needing preprocessing.

Right-skewed. Mean=₹3,999 >> Median=₹619. Histogram: most expenses cluster at ₹100-500, tail stretches to ₹1,01,000.
Web server response times = right-skewed. Most requests: 50ms. Rare tail-latency requests: 5000ms. SLA is set at the 99th percentile (the right tail).
Always plot distribution before modelling. Right-skewed data needs preprocessing (log transform, robust scaling) for algorithms that assume normality.
Correlation (r)
Do two variables move together? How strongly?
StatisticsPhase 1Phase 3
r ranges from -1 to +1: r = +1.0 perfect positive (x up → y always up) r = +0.5 moderate positive (x up → y usually up) r = 0.0 no relationship r = -0.5 moderate negative (x up → y usually down) r = -1.0 perfect negative (x up → y always down) Three patterns: Positive Negative None * * * * * * * * * * * * * * * * * * x increases → y up y down no pattern
r = Σ((xᵢ-x̄)(yᵢ-ȳ)) / (n × std_x × std_y) r² = R-squared = % of variance explained by the model
All categories: r=0.485 (Savings inflate upward trend)
Exclude Savings: r=0.298 (real outflows, weak positive)
Exclude Savings+LOANS: r=-0.262 (pure discretionary, slightly decreasing)

Insight: your EMI/CC is growing over time but day-to-day spending is leaner.
Confounding: Savings entries were masking the real trend. Remove them → r flipped direction. Always understand what's in your data before modelling.
Correlation measures linear relationship strength. r² (R-squared) tells you what % of variance your model explains. Always check correlation before regression — if r≈0 there's nothing to model.
numpy vs Python Loops
Vectorised operations — why ML is computationally possible
EngineeringAll Phases 
# Python loop — one number at a time
total = 0
for a in df['amount']:
    total += a          # 840 iterations

# numpy — entire array in one CPU instruction
total = np.sum(df['amount'].values)   # 1 operation
Benchmark: Python 0.365ms vs numpy 0.159ms = 2× on 840 rows. At 1,000,000 rows → ~100× faster. Neural net training = millions of rows × thousands of passes.
Paying 840 employees. Loop = walk to each desk one by one. numpy = one bank transfer that auto-splits. Same result, completely different scale.
Vectorised operations use CPU SIMD instructions, avoiding Python's interpreter overhead. For ML workloads this is the difference between minutes and hours of training.
