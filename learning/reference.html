Vector
A list of numbers representing one thing
Linear AlgebraAll Phases
One expense row = one vector ┌────────────┬────────┬──────────┬─────────┐ │ 20260501 │ 450.0 │ 0 │ 1 │ └────────────┴────────┴──────────┴─────────┘ date amount category payment ◄──────────────── 4 dimensions ────────────►
vector = [x₁, x₂, x₃, ... xₙ] ← n values, n dimensions
Analogy: Your Swiggy receipt. One purchase described by multiple values — date, amount, category, payment method. Convert each to a number → that list IS a vector.
Google Search converts your query into a vector of numbers. Results whose vectors are closest to yours get ranked higher. Same math.
A vector is an ordered list of numbers representing a point in n-dimensional space. In ML, one row of a dataset is called a feature vector.

shape = (rows, columns) = (n_samples, n_features)
Analogy: Your expenses spreadsheet. Every row is one expense (vector). Stack 837 of them → a matrix. Every ML algorithm is just math operations on this matrix.
A 100×100 grayscale image = a (100, 100) matrix where each number is pixel brightness. Neural networks read images as matrices directly.
A matrix is a 2D array with shape (m, n). Training data is represented as matrix X of shape (n_samples, n_features). All linear algebra operations in ML run on matrices.
Dot Product
Multiply element-by-element, then sum. The core operation of all ML.
Linear AlgebraPhase 1Phase 5Phase 6
Step 1 — multiply element-by-element: a = [ 1, 2, 3 ] × × × b = [ 4, 5, 6 ] = = = [ 4, 10, 18 ] Step 2 — sum: 4 + 10 + 18 = 32
a · b = a[0]×b[0] + a[1]×b[1] + ... + a[n]×b[n]
Linear regression IS a dot product: features = [month, prev_spend, season ] weights = [ w1, w2, w3 ] prediction = features · weights = (month × w1) + (prev_spend × w2) + (season × w3) The algorithm's only job: find the best weights vector.
Similarity — what the dot product measures: a = [10, 10], b = [10, 10] → a·b = 200 (same direction = very similar) a = [10, 0], b = [ 0, 10] → a·b = 0 (perpendicular = nothing in common) Large dot product → vectors are similar Zero dot product → nothing in common
Netflix recommendation: your_taste · movie_profile = match score. Higher dot product = better match. Recommendation engines are dot products at their core.
Dot product is the fundamental operation in ML. Linear regression, neural network layers, and transformer attention are all dot products. It measures alignment (similarity) between two vectors.
Mean & Median
Two ways to describe the centre of your data
StatisticsPhase 1Phase 2
Mean
mean = sum / count = (100+200+300+400+500) / 5 = 300
Sensitive to outliers. One large EMI pulls it up.
Median
Sort → pick middle value [100, 200, 300, 400, 500] median = 300
Ignores outliers. Robust to skew.
Mean = ₹3,999 vs Median = ₹619. Gap is huge because EMI/CC payments pull mean up. Median tells the real story — most day-to-day expenses are under ₹619.
Analogy: 5 friends' salaries — ₹3L, ₹4L, ₹5L, ₹6L, ₹82L. Mean = ₹20L (looks rich). Median = ₹5L (actual reality). One outlier destroyed the mean.
City salary reports always use median, not mean. A few tech executives inflate the mean. Median gives the typical worker's picture.
When mean >> median, data is right-skewed. For skewed distributions, median is a better measure of central tendency. Always check both before reporting.
Variance & Standard Deviation
How spread out is the data from the centre?
StatisticsPhase 2Phase 5
Given: [100, 200, 300, 400, 500], mean = 300 Step 1 — distance from mean: 100-300 = -200 200-300 = -100 300-300 = 0 400-300 = +100 500-300 = +200 Step 2 — square each (removes negatives, punishes large gaps): 40000 + 10000 + 0 + 10000 + 40000 = 100000 Step 3 — average: variance = 100000 / 5 = 20000 (in ₹²) Step 4 — square root to get back to ₹: std dev = √20000 ≈ 141 (in ₹)
variance = Σ(xᵢ - mean)² / n std dev = √variance
The 68-95-99 rule (critical for Phase 2 anomaly detection): ┌──────────────────────────────────────────────┐ │ ███ │ │ ███████ │ │ 68% ███████████ │ │ ◄──────► ██████████████ │ │ 95% ████████████████ │ │ ◄────────► ██████████████████ │ │ mean-2σ mean mean+2σ │ └──────────────────────────────────────────────┘ 68% of data → within ±1 std dev of mean 95% of data → within ±2 std dev of mean 99.7% data → within ±3 std dev of mean
Mean=₹3,999, Std=₹13,301. Std larger than mean = extreme spread. 68% of expenses fall between ₹0 and ₹17,300. The distribution is very spread out due to EMI/CC entries.
Two cricket batsmen both average 50. Batsman A always scores 45-55 (low std dev = consistent). Batsman B scores 0 or 100 randomly (high std dev = unpredictable). Same mean, very different std dev.
Stock volatility IS std dev. Stable blue-chip stock = low std dev. Meme stock = high std dev. In manufacturing, bolt diameter tolerance = std dev specification.
Std dev is √variance, expressed in original units. Used in Z-score (Phase 2), confidence intervals, and neural net weight initialisation. High std dev relative to mean signals skewed data needing preprocessing.

The shape of your data — where values cluster and how spread out they are

Your spending is right-skewed. Tall bars on the left (most expenses ₹100-₹500). Long tail stretching right to ₹1,01,000 (EMIs, CC). Frequency y-axis peaks near 600 at low amounts.
Exam scores in a class → usually normal (bell curve, most around 60-70). House prices in a city → right-skewed (most affordable, few luxury mansions create the right tail).
Web server response times are right-skewed. Most requests: 50ms. A few "tail latency" requests: 5000ms. Your 99th percentile SLA is set based on that right tail.
Always plot your distribution before modelling. Right-skewed data needs preprocessing (log transform, robust scaling) for algorithms that assume normality (Z-score, linear regression).
numpy vs Python Loops
Vectorised operations — the reason ML is computationally possible
EngineeringAll Phases 
# Python loop — one number at a time (slow)
total = 0
for a in df['amount']:       # 840 iterations
    total += a

# numpy — entire array at once (fast)
total = np.sum(df['amount'].values)   # 1 operation
Benchmark on your 840 expenses: Python loop 0.365ms vs numpy 0.159ms = 2× faster. At 1,000,000 rows → ~100× faster. Neural net training runs this millions of times.
Paying 840 employees. Loop = walk to each desk one by one. numpy = one bank transfer that auto-splits. Same result, completely different scale.
pandas, scikit-learn, TensorFlow, PyTorch — all built on numpy. When you call model.fit(), it's running optimised matrix operations, not Python loops.
Vectorised operations avoid Python's interpreter overhead and use CPU SIMD instructions. For ML workloads this is the difference between minutes and hours of training time.
